# OOXML — Inserting Worksheets from a Base64 XLSX File

## Key Types
- `context.workbook.insertWorksheetsFromBase64(base64File, options?)` — inserts one or more worksheets from a base64-encoded XLSX file into the current workbook. Returns a `WorksheetCollection` of the newly inserted sheets.
- `WorksheetInsertOptions` — plain object controlling how sheets are inserted:
  - `positionType?` — `Excel.WorksheetPositionType`: `none`, `before`, `after`, `beginning`, `end`. Defaults to `end` if omitted.
  - `relativeTo?` — a `Worksheet` object; required when `positionType` is `before` or `after`.
  - `sheetNamesToInsert?` — string array of sheet names to pull from the source file. Pass an empty array or omit entirely to insert ALL sheets from the file.
- `Excel.WorksheetPositionType` — enum: `none`, `before`, `after`, `beginning`, `end`.

---

## When to Use

Use `insertWorksheetsFromBase64` when you need to:
- Copy a template sheet with rich formatting, merged cells, charts, or custom styles that are difficult to replicate cell-by-cell via the API.
- Import a pre-built XLSX that was generated server-side.
- Seed a workbook with one or more formatted starter sheets in a single API call.

---

## Base64 String Requirements

The source must be a complete, valid XLSX package. XLSX is a ZIP archive containing at minimum:
- `[Content_Types].xml`
- `_rels/.rels`
- `xl/workbook.xml`
- `xl/worksheets/sheet1.xml`
- `xl/_rels/workbook.xml.rels`

The base64 string MUST be the raw encoding of that ZIP — do NOT include a data-URL prefix such as `data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,`. Strip any such prefix before passing the string.

Typically the XLSX is generated by a server or build step and delivered to the add-in as a plain base64 string.

---

## Reading the Names of Inserted Sheets

`insertWorksheetsFromBase64` returns a `WorksheetCollection`. Load `items/name` (or other properties) to inspect the results.

```javascript
await Excel.run(async (context) => {
  const inserted = context.workbook.insertWorksheetsFromBase64(base64String);

  inserted.load("items/name");
  await context.sync();

  for (const sheet of inserted.items) {
    console.log("Inserted sheet:", sheet.name);
  }
});
```

## Example 1 — Insert All Sheets from a Base64 XLSX at the End of the Workbook

```javascript
await Excel.run(async (context) => {
  // base64String is a raw base64-encoded XLSX — no data-URL prefix.
  const inserted = context.workbook.insertWorksheetsFromBase64(base64String, {
    positionType: Excel.WorksheetPositionType.end,
    // sheetNamesToInsert omitted — all sheets in the file are inserted.
  });

  inserted.load("items/name");
  await context.sync();

  console.log("Sheets inserted at end of workbook:");
  for (const sheet of inserted.items) {
    console.log(" -", sheet.name);
  }
});
```

---

## Example 2 — Insert One Named Sheet BEFORE the Active Sheet

```javascript
await Excel.run(async (context) => {
  const activeSheet = context.workbook.worksheets.getActiveWorksheet();

  // Insert only the sheet named "Template" from the source XLSX,
  // positioned before the currently active sheet.
  const inserted = context.workbook.insertWorksheetsFromBase64(base64String, {
    positionType: Excel.WorksheetPositionType.before,
    relativeTo: activeSheet,
    sheetNamesToInsert: ["Template"],
  });

  inserted.load("items/name");
  await context.sync();

  console.log("Inserted:", inserted.items[0].name);
});
```

## Common Mistakes

- **Do not include the data-URL prefix**: `insertWorksheetsFromBase64` expects raw base64. If you pass `"data:application/...;base64,<data>"`, the runtime cannot parse the XLSX and will throw. Strip the prefix: `base64String = dataUrl.split(",")[1]`.
- **Conflicting sheet names are auto-renamed**: If the source sheet is named `"Sheet1"` and the workbook already contains a sheet with that name, Excel renames the inserted sheet automatically (e.g. `"Sheet1 (2)"`). Load `items/name` after sync to read the actual assigned name.
- **Invalid XLSX packages throw `InvalidArgument`**: The base64 content must decode to a structurally valid XLSX ZIP. A corrupt archive, a CSV, or a binary that was incorrectly base64-encoded will all throw at sync time.
- **Large files block the sync**: `insertWorksheetsFromBase64` sends the entire base64 string in a single message to the Excel host. For very large files this can be slow or time out. If the source file is large, generate a minimal XLSX server-side (only the sheets and data you need) rather than sending the full workbook.
- **`before`/`after` require `relativeTo`**: If you set `positionType` to `before` or `after` without supplying a `relativeTo` worksheet reference, the call will throw. Make sure the `Worksheet` object passed as `relativeTo` belongs to the current workbook context.
- **`sheetNamesToInsert` is case-sensitive**: The names in the array must exactly match the sheet tab names in the source XLSX. A mismatch silently inserts nothing (or throws, depending on the API version).
